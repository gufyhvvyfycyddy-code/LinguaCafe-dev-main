# LinguaCafe Dev Main 开发工作流

## 仓库定位

- 本仓库是新开发主线仓库。
- 原仓库（LinguaCafe-local）作为历史仓库，不再自动推送。
- 后续开发完成一轮并通过网页端 GPT 验收后，应 push 到本仓库。

## 每次开发任务前

1. 网页端 GPT 必须基于 GitHub 最新代码进行分析。
2. 本地 Agent 只执行当前任务，不凭历史聊天记录判断代码状态。

## 每轮执行后报告格式

完成一轮开发任务后，必须向用户报告以下内容：

- git status — 当前改动状态
- 修改了哪些文件 — 文件列表
- 测试结果 — phpunit 输出摘要
- 浏览器 smoke — 手动验证的关键页面
- 是否已 push 到本仓库

## 用户确认流程

1. 用户确认 Accept 后，下一轮继续。
2. 未经验收不得 push。
3. 后续开发不要再默认 push 旧仓库。

## 禁止事项

- 不要凭聊天记录判断代码状态 —— 以 git status 和实际文件为准。
- 不要凭预期行为判断完成 —— 必须通过测试和浏览器验证。
- 不要自动执行 DCP。
- 不要运行 notification script。
- 不要运行 migrate:fresh、db:wipe。
